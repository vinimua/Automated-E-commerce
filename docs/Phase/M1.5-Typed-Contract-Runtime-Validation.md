> 目标：把 M1 阶段的“文档契约”补强为“运行时契约”，让 Python AI 输出、RenderManifest、前端表单输入和跨文件枚举在开发期/运行期更早暴露问题。

## 2. 解决的问题

| 问题 | 原风险 | 当前方案 |
|---|---|---|
| AI 输出只靠文档约束 | 大模型容易返回多字段、少字段、字段名错，错误会一路传到 Java | Python Pydantic 在回调前校验；失败会在 AI Orchestrator 内暴露 |
| Fashion 新链路没有运行时模型 | asset/reference/keyframe/video_clip/qa/repair 没有强类型边界 | 补齐 Fashion Pydantic models，并挂到 `CallbackPayload` |
| RenderManifest 手写校验和 TS 类型分离 | 修改字段时容易只改类型或只改校验 | 用 Zod schema 同时承载类型推导和运行时校验 |
| 前端表单缺少统一校验入口 | 页面各自写校验，字段规则容易漂移 | 新增可复用 Zod schema，后续页面可直接接 `react-hook-form` |
| 多份契约容易漂移 | DB/OpenAPI/Python/Render 枚举改一处漏一处 | 新增 `scripts/check-contract-sync.ps1` 做开发期检查 |

## 3. 运行时约束范围

### 3.1 Python AI 输出

已覆盖：

- 旧链路：`ProductAnalysis`、`VideoPlanResult`、`StoryboardResult`、`MaterialResult`、`QualityCheckResult`
- Fashion 新链路：`FashionAssetAnalysis`、`ReferenceVideoAnalysis`、`CreativePlanResult`、`KeyframeGenerationResult`、`VideoClipGenerationResult`、`RepairResult`、`FashionQaResult`
- 回调载荷：`CallbackPayload`

关键规则：

- 所有模型默认拒绝额外字段。
- `status = failed` 时必须有 `error`。
- `status = success` 时，按 `stage` 要求对应 payload 字段。
- 关键生成项 `completed` 时必须有 `url`；`failed` 时必须有 `errorMessage`。

### 3.2 RenderManifest

已覆盖：

- `manifestVersion = 1.0.0`
- `taskId/videoId` UUID
- `videoType/template/resolution/fps/duration`
- `assets` 数量、镜头号唯一性、文本素材和非文本素材的条件字段
- `subtitleStyle/music/voiceover/cover/output`
- 总时长和镜头时长误差不超过 1 秒
- `cover.sourceShotNo` 必须对应真实镜头

### 3.3 前端 Zod Schema

已新增：

- `video-task.ts`
- `task-asset.ts`
- `keyframe.ts`
- `video-clip.ts`
- `feedback.ts`
- `index.ts`

当前只提供 schema，不强制改造现有页面。M3 实现新页面时应优先复用这些 schema。

## 5. 后续建议

| 优先级 | 建议 | 说明 |
|---|---|---|
| P0 | 在 CI 中加入 `scripts/check-contract-sync.ps1` | 防止后续 AI 编码或人工改动导致契约漂移 |
| P1 | M2 Java 实现时同步新增 Enum/DTO/StateMachine | 当前 Python/OpenAPI/DB 已准备好，Java 运行时还需落地 |
| P2 | M3 前端页面接入 `apps/web/src/schemas` | 表单提交前先 Zod 校验，再调用 Java API |
| P3 | 为 Zod/Pydantic 增加样例测试 | 用正例/反例固定关键契约行为 |
| P4 | 长期考虑自动生成 | 例如 Pydantic -> JSON Schema、OpenAPI -> TS types，但短期先保持手写可控 |

---

## 6. 结论

M1.5 已把项目从“主要依赖文档说明”推进到“关键边界具备运行时 schema”。当前最核心的收益是：AI 输出和 RenderManifest 不合法时，会在服务边界被拒绝，而不是继续流入 Java、数据库或渲染流程。
